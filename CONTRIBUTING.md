# Contributing to Dynamic Narrative Director

Thanks for your interest in contributing! This project aims to build an emotionally intelligent narrative framework.

## Development Workflow
- Fork and create feature branches from `main`.
- Write tests for new features.
- Ensure linting passes: `npm run lint`.
- Run tests locally: `npm test`.
- Update docs where appropriate.

## Commit & PR Guidelines
- Use clear, descriptive commit messages.
- Keep PRs focused and small.
- Include context and reasoning in PR description.
- Link related issues.

## Code Style
- TypeScript strict mode enforced.

## Security & Privacy
- Do not commit secrets; use env vars.
- Follow least-privilege and sanitize external inputs.
